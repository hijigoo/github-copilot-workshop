## Chat 패널 UI 구성

### 입력창 구성

Chat 입력창 주변에는 다음 요소들이 있습니다:

| 요소 | 설명 |
|------|------|
| 파일 태그 (예: `main.py`) | 현재 참조 중인 파일. "끄기"를 클릭하면 제외할 수 있습니다 |
| 입력창 | `@` 참가자, `#` 채팅 변수, `/` 슬래시 명령어를 사용할 수 있습니다 |
| 모드 선택기 (예: `Agent`) | 좌측 하단 드롭다운. Ask, Agent, Plan 중 선택합니다 |
| 모델 선택기 (예: `Claude Sonnet 4.6`) | 응답에 사용할 AI 모델을 변경할 수 있습니다 |
| 🔧 (도구 아이콘) | MCP 서버 등 외부 도구(Tool)를 연결하고 관리합니다. Agent 모드에서 활용됩니다 |

### 모드 선택 (agents 드롭다운)

입력창 좌측 하단의 **agents 드롭다운**을 클릭하면 모드를 선택할 수 있습니다:

| 모드 | 설명 |
|------|------|
| **Ask** | 질문/답변 모드. 코드를 직접 수정하지 않고 답변만 합니다. |
| **Agent** | 자율적으로 파일 생성/수정/삭제 + 터미널 실행까지 수행합니다.|
| **Plan** | 코드 변경 전에 상세한 구현 계획을 먼저 세우고, 검토 후 실행합니다. |

## `@`, `#`, `/` — Chat 키워드 기본 개념

Chat 입력창에서 세 가지 특수 기호를 조합하여 Copilot과 대화합니다.

| 기호 | 역할 | 설명 | 예시 |
|------|------|------|------|
| `@` | **참가자** (chat participants) | 특정 도메인 전문가를 호출합니다 | `@github`, `@terminal` |
| `#` | **채팅 변수** (chat variables) | 특정 파일, 폴더 등을 컨텍스트로 전달합니다 | `#file:app/main.py` |
| `/` | **슬래시 명령어** (slash commands) | 자주 쓰는 작업을 간단히 지시합니다 | `/explain`, `/fix`, `/tests` |

### `@` 참가자 (Chat Participants)

참가자는 특정 도메인의 전문가 역할을 합니다. Copilot은 자연어 프롬프트를 분석하여 적절한 참가자를 **자동으로 추론**할 수도 있고, `@`를 입력하여 직접 지정할 수도 있습니다.

| 참가자 | 설명 |
|------|------|
| `@github` | GitHub 스킬을 활용합니다. 웹 검색(`#web`), 코드 검색 등이 가능합니다 |
| `@terminal` | VS Code 터미널 셸과 그 내용에 대한 컨텍스트를 가집니다. 터미널 명령어 작성이나 디버깅에 활용합니다 |
| `@vscode` | VS Code 명령어와 기능에 대한 컨텍스트를 가집니다. VS Code 설정 관련 질문에 활용합니다 |

> 💡 `@`를 입력하면 사용 가능한 모든 참가자 목록이 표시됩니다.

### `#` 채팅 변수 (Chat Variables)

채팅 변수를 사용하면 특정 컨텍스트를 프롬프트에 포함할 수 있습니다.

| 변수 | 설명 |
|------|------|
| `#file` | 특정 파일의 내용을 컨텍스트로 전달합니다 |
| `#selection` | 현재 선택된 텍스트를 전달합니다 |
| `#block` | 현재 코드 블록을 전달합니다 |
| `#class` | 현재 클래스를 전달합니다 |
| `#function` | 현재 함수 또는 메서드를 전달합니다 |
| `#sym` | 현재 심볼을 전달합니다 |
| `#project` | 프로젝트 컨텍스트를 전달합니다 |
| `#path` | 파일 경로를 전달합니다 |

> 💡 `#`를 입력하면 사용 가능한 모든 채팅 변수 목록이 표시됩니다.

### `/` 슬래시 명령어 (Slash Commands)

| 명령어 | 설명 |
|--------|------|
| `/explain` | 선택한 코드 또는 현재 파일의 동작을 설명합니다 |
| `/fix` | 선택한 코드의 문제를 분석하고 수정을 제안합니다 |
| `/tests` | 선택한 코드에 대한 단위 테스트를 생성합니다 |
| `/new` | 새 프로젝트를 생성합니다 |
| `/newNotebook` | 새 Jupyter 노트북을 생성합니다 |
| `/clear` | 새 채팅 세션을 시작합니다 |

> 💡 `/`를 입력하면 사용 가능한 모든 슬래시 명령어 목록이 표시됩니다.

## 태스크 A: `@` — 프로젝트 맥락 질문 (3분)

Chat에 입력:

```
@terminal #terminalLastCommand 마지막 명령어와 에러를 설명해줘
```
**관찰**: `@terminal` 참가자가 터미널의 마지막 명령어와 에러 메시지를 읽고 분석하여 설명합니다.

---

## 태스크 B: `#` — 코드 설명 받기 (3분)

Chat에 입력:

```
/explain #file:app/main.py 한국어로 설명해줘
```

**관찰**: `#file:`로 파일을 지정하면 Copilot이 해당 파일의 구조와 각 엔드포인트의 역할을 한국어로 설명합니다.


---

## 태스크 C: `/fix` — 버그 수정 (5분)

1. `app/main.py`의 `create_todo` 함수에서 **의도적 버그** 삽입:

```python
# 이 부분을 일부러 잘못 고치세요:
new_todo = {
    "id": next_id,
    "title": todo.description,   # ← 버그! title이 아니라 description
    "completed": True,            # ← 버그! 기본값이 True
}
```

2. 버그가 있는 코드를 선택하고 Agent 모드 Chat에 입력:

```
/fix
```

**관찰**: Copilot이 잘못된 필드 매핑과 기본값을 정확히 찾아냅니다.

---

## 태스크 D: `/tests` — 테스트 코드 생성 (5분)

Agent 모드 Chat에 입력:

```
/tests #file:app/main.py
```

💡 자동으로 파일이 생성되지 않으면, 이어서 다음과 같이 입력하세요:
 ```
테스트 파일을 만들고 실행해줘
```
**관찰**: Copilot이 `tests/test_main.py` 파일을 생성하며, httpx + pytest 기반 테스트를 작성합니다.

/fix #file:app/main.py

```
이 에러 수정해줘:
pytest에서 AssertionError가 나는데, 응답 필드가 expected와 다르대
```

## 핵심 인사이트

- **`#file:`로 범위를 좁혀라** — 프로젝트 전체를 보여주면 답이 흐려집니다. 관련 파일만 지정하면 정확도가 올라갑니다
- **모드를 구분하라** — 질문은 Ask, 코드 수정은 Agent. 목적에 맞는 모드를 선택하세요
- **`@`, `#`, `/`를 조합하라** — 참가자(`@`) + 채팅 변수(`#`) + 슬래시 명령어(`/`)를 조합하면 한 줄로 복잡한 작업을 지시할 수 있습니다
- **에러도 Chat으로 해결하라** — `/fix`와 에러 메시지 붙여넣기만으로 대부분의 문제가 해결됩니다
